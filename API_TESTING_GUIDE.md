## Phase 2: Authentication - OTP Flow

### 2.1 Send OTP (New User Registration)
```bash
POST /api/auth/send-otp

Body: {
  "phoneNumber": "+919999000001"
}

Expected: 200 OK
Response: {
  "success": true,
  "message": "OTP sent successfully",
  "data": {
    "expiresIn": "10 minutes",
    "challengeToken": "eyJ...", // JWT token
    "otp": "123456" // Only in development!
  }
}

Note: In production, OTP is not returned
```

### 2.2 Register & Verify OTP (New User)
```bash
POST /api/auth/verify-otp-register

Body: {
  "phoneNumber": "+919999000001",
  "otp": "123456",
  "fullName": "John Doe",
  "deviceId": "device-uuid-001",
  "deviceInfo": "iPhone 12 Pro",
  "challengeToken": "eyJ..." // From step 2.1
}

Expected: 201 Created
Response: {
  "success": true,
  "message": "User registered successfully",
  "data": {
    "user": {
      "id": "user-uuid",
      "phoneNumber": "+919999000001",
      "fullName": "John Doe",
      "role": "CUSTOMER"
    },
    "accessToken": "eyJ...",
    "refreshToken": "eyJ..."
  }
}

Errors to Check:
- 400: Missing fields
- 400: Invalid OTP
- 409: User already exists
```

### 2.3 Login with OTP (Existing User)

## Test Automation Script

```bash
#!/bin/bash
# save as: test-api.sh

BASE_URL="http://localhost:3001"
CUSTOMER_PHONE="+919999000001"
OTP="123456"

echo "🧪 Running API Tests..."
echo ""

echo "1️⃣  Health Check..."
curl -X GET "$BASE_URL/health"
echo ""
echo ""

echo "2️⃣  Send OTP..."
OTP_RESPONSE=$(curl -s -X POST "$BASE_URL/api/auth/send-otp" \
  -H "Content-Type: application/json" \
  -d "{\"phoneNumber\": \"$CUSTOMER_PHONE\"}")

CHALLENGE_TOKEN=$(echo $OTP_RESPONSE | grep -o '"challengeToken":"[^"]*' | cut -d'"' -f4)
echo "Challenge Token: $CHALLENGE_TOKEN"
echo ""

echo "3️⃣  Register & Verify OTP..."
REGISTER_RESPONSE=$(curl -s -X POST "$BASE_URL/api/auth/verify-otp-register" \
  -H "Content-Type: application/json" \
  -d "{
    \"phoneNumber\": \"$CUSTOMER_PHONE\",
    \"otp\": \"$OTP\",
    \"fullName\": \"Test User\",
    \"deviceId\": \"test-device-001\",
    \"deviceInfo\": \"Test\",
    \"challengeToken\": \"$CHALLENGE_TOKEN\"
  }")

echo $REGISTER_RESPONSE | jq '.'
ACCESS_TOKEN=$(echo $REGISTER_RESPONSE | grep -o '"accessToken":"[^"]*' | head -1 | cut -d'"' -f4)
echo "Access Token: $ACCESS_TOKEN"
echo ""

echo "4️⃣  Get Current User..."
curl -s -X GET "$BASE_URL/api/customers/me" \
  -H "Authorization: Bearer $ACCESS_TOKEN" | jq '.'
echo ""

echo "✅ Tests Complete!"
```

## Error Response Format

All errors follow this format:

```json
{
  "success": false,
  "message": "Error description",
  "error": {
    "code": "ERROR_CODE",
    "details": "Additional details"
  }
}
```

### Common Error Codes
- `VALIDATION_ERROR` - Invalid input
- `AUTH_ERROR` - Authentication failed
- `NOT_FOUND` - Resource not found
- `CONFLICT` - Resource already exists
- `SERVER_ERROR` - Unexpected error

## Debugging Tips

**OTP not received:**
- Check `backend/src/utils/envValidator.ts` for SMS provider status
- Verify Twilio credentials configured
- Check backend logs for SMS errors

**Authentication failed:**
- Verify JWT_SECRET and JWT_REFRESH_SECRET are set
- Check token hasn't expired
- Verify Authorization header format: `Bearer <token>`

**Database errors:**
- Ensure PostgreSQL is running
- Run: `npm run prisma:migrate` to apply migrations
- Check DATABASE_URL connection string

**CORS errors:**
- Check frontend origin in `.env`
- Update CORS_ORIGIN in backend config
